Intro - 什么是git?

git被官方解释为 : 分布式版本控制器(Distributed Version Control System)。根据这一点，我会从以下几点去解释:

  1. 什么是版本控制(Version control)？
  2. 为什么使用分布式结构(Distributed structure)
  3. 为什么使用git?

Version control

def : 版本控制关注的是文件，代码。当个人/团队需要完成一项工程时，我们关注的是这项工程中的每份文件的进度。通过版本控制系统，每次对内容上的修改都会通过以下方式记录:

  • 修改时间
  • 负责人
  • 版本编号

当某个git(这里翻译为bastard: 小混蛋, 俚语)用任何一种方式毁了团队的项目，版本控制器提供的以上记录可以被翻译为:

  • 修改时间: 找到作案时间
  • 负责人: 抓到嫌疑人
  • 版本编号: 回溯到之前版本

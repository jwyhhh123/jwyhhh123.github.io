Set Theory

WY J
2019-03-11 (Updated: 2019-06-17)
Year1/ Mathematics Fundation of Computer Science

In set theory, we will discuss the following topics :

  • Sets
  • Power Sets
  • Relations

Sets

The intuitive idea of a set

def : a set is a collection of things. And, the things in the set are called elements/ members.

  • The elements of a set are identifiable and can be distinguished.
  • There is a clear criterion that identify members.
  • A member must be counted ONLY once.
  • there is no order of members.
  • if two sets have the same elements, then they form the same set, independently of how the collections were defined.

NOTE : Set in JAVA is supported as an interface and one possible implementation is HashSet.

Notation

  1. To indicate x is an element in set A, we write : x ∈ A
  2. A = B iff ∀x. x∈A <-> x∈B
  3. Finite sets : {Spring, Summer, Autumn, Winter}
  4. Infinite sets : {1, 4, 9, 16, 25, …}

Sets of numbers

1) Union of two sets : ∪

  • Integers can be represented : Z = N ∪ {−1,−2,−3,...}

2) Product of two sets :

  • if A and B are different, A x B
  • if A = B, A x B = A ^ 2

3) set difference : \

  • The set of fractions could be written as : Z x (Z \ {0})

4) Subset : A ⊆ B

  • Q (rational numbers) is a subset of the set of fractions.

5) the tuples in products :

  • B x B x…x B in 32-tuples : B^32 = {0, 1}^32
  • assume A is an alphabet set of {a, …, z}, then A^* is the set of all words over A.
  • in JAVA, Unicode is the set of alphabet.
  • Real numbers can be represented as : R = Z x D^w, where D = {0, 1, …, 9}

NOTE : a finite ring Zm = {0, 1, …, m-1} is viewed as the set of integers with alternate equality “congruent modulo m”.

Counting the elements of a set

size of a set can be written as card(k) or |k|
card(k) : cardinality
 
Instead of counting, we check two sets have the same cardinality by pairing one element by one element between the two sets.
 
example : card(Seasons) = card(Compass)

Cardinality of infinite sets

A set is countably infinite if it has the same cardinality as N :

  • A = N ∪ {−1} is countable infinite set
  • Z countable infinite set
  • N^2 is countable infinite set
  • if A is finite set then A^* is countable
  • Theorem 7 : The set of valid Java programs is countable

An uncountable set

real number R is an uncountable set
 l;
proof steps :

  1. assume R is countable
  2. given a real number : a = a0 a1 a2 …
  3. list some real numbers in a matrix
  4. for i(s), ai = 0 if r(i,i) != 0
  5. for i(s), ai = 1 if r(i,i) == 0
  6. we will introduce a real number have not appear in the matrix.
  7. the matrix is not complete.
  8. contradiction, hence the set of real numbers is uncountable.

Theorem 8 : The set of real numbers is uncountable
Theorem 9 : There are non-computable real numbers (as set of java programs is countable and set of reals is uncountable)

Comparing uncountable sets
The examples above illustrate how we can establish that two countably infinite sets have the same cardinality. To show that two uncountable sets have the same size requires a bit more ingenuity but the principle is the same: We establish a one-to-one relationship between the elements. For example, we can show that the elements of a circle (minus one point) can be lined up against the elements of the real line.

Power Sets and Boolean Algebra

Defining a subset

If subset has only few elements, we can list them : {3, 5, 17, 257, 65537} ⊆ N
A set excluding some members is a subset of the original set : F\{0} ⊆ F where F is a field.
Empty set is a subset of any set : {} or 0/ (zero with a slash)
A set is a subset of itself : A ⊆ A
For more complex subsets we employ logical conditions : B = {x ∈ A | x satisfies condition c}

Comparing subsets

B ⊆ A : B ⊆ A iff ∀x. x ∈ B –> x ∈ A
equality of two sets : A = B iff A ⊆ B and B ⊆ A
 
NOTE : there are two conditions to check equality: firstly A ⊆ B, secondly B ⊆ A

Operations on subsets

Union : B1 ∪ B2 = {x ∈ A | x ∈ B1 or x ∈ B2}
Intersection : B1 ∩ B2 = {x ∈ A | x ∈ B1 and x ∈ B2}
Complement : -B = {x ∈ A | x !∈ B}
Set difference : B1 \ B2 = B1 ∩ -B2 = {x ∈ A | x ∈ B1 and x !∈ B2}

Power sets

def : the collection of the subsets of a set A. It is written as PA

  • the elements of a power set are identifiable(for each is different from others).
  • if B is a member of PA then B is a subset of A
  • we ALWAYS have empty subset as an element in PA.

there are ALWAYS more members in PA than in A.

Theorem 10 : for any set A, the cardinality of the power set PA is strictly larger than the cardinality of A.

NOTE : the powerset of N is uncountable due to theorem 10.

Relations

General relations

def : if A1, A2, …, An are sets and R is a subset of A1 x A2 x …x An, then R is called a (n-ary) relation.
 
NOTE : the elements of A1 ×A2 ×…×An are n-tuples (x1,x2,…,xn). Also, note that any subset is allowed; there are no conditions for a subset to be called a relation.
 
Example : 3-ary relation given A1 : the set Σ∗ of strings, and A2, A3: sets of N

NOTE : we are looking ar the special case of 2-ary(binary) relations on a single set. To explain, we look at subsets of A x A
 
Directed graph representation :

  • vertices : elements in A
  • edges : relations/ subsets of A

Reflexivity

def : We call a binary relation R ⊆ A × A reflexive if it contains all pairs (x, x) for x an element of A.
 
R ⊆ A x A is reflexive = ∀x ∈ A. (x, x) ∈ R
 
if given a relation R that is not reflexive, then we can make it reflexive by adding the missing edges :
refl-closure(R) = R ∪ {(x, y) ∈ A x A | x = y}
 
Irreflexive relation : relation that contains no edge pointing to itself

Symmetry

def : A binary relation R ⊆ A × A is called symmetric if with every pair (x, y) ∈ R we also have (y, x) ∈ R.
 
R ⊆ A × A is symmetric = ∀(x,y) ∈ A×A. (x,y) ∈ R -> (y,x) ∈ R
 
if given a relation R that is not symmetric, then we can make it symmetric by giving:

  1. inverse relation : R^(-1) = {(x,y) ∈ A x A | (y,x) ∈ R}
  2. symm-closure(R) = R ∪ R^(-1)

Anti-symmetric : there is no a and b connecting each other. While reflexivity is allowed.

Transitivity

def : A binary relation R ⊆ A × A is called transitive if whenever (x, y) ∈ R and (y, z) ∈ R then also (x, z) ∈ R.
 
R ⊆ A × A is transitive = ∀x,y,z ∈ A.(x,y) ∈ R and (y,z) ∈ R -> (x,z) ∈ R
 
if given a relation R that is not transitive, then we can make it transitive by given :

  1. composition of binary relations : R;S = {(x,z) ∈ A x A | ∃y ∈ A. (x, y) ∈ R and (y, z) ∈ S}
  2. trans-closure(R) = R ∪ (R;R) ∪ (R;R;R) ∪ …

NOTE : this idea is like finding a path from liverpool to birmingham, there are lot of paths we can choose.

Two packages of properties

Order relation :
A relation R ⊆ A × A is called an order relation if it is reflexive, anti-symmetric, and transitive.

Equivalence relation :
A relation R ⊆ A × A is called an equivalence relation if it is reflexive, symmetric, and transitive.

Equivalence relations

symbol : ≈ (congruence modulo m is commonly written as ≡ or ≡m).
equiv-closure(R) = trans-closure(symm-closure(refl-closure(R)))

Equivalence relations lead to classifications

Equivalence classification : [a]≈ = {x ∈ A| x ≈ a} where a is an element of A.
 
Theorem : if ≈ is an equivalence relation on a set A, and a, b are elements of A, then always one of the following two situations is true :

  1. a ≈ b and [a]≈ = [b]≈
  2. a !≈ b and [a]≈ ∩ [b]≈ = 0/

NOTE : this means class [a]≈ and class [b]≈ are disjointed.

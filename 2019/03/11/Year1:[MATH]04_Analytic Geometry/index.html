Practical advice

NOTES:

  • Always write the coordinates of points and vectors in the column form. This will help you to avoid mistakes such as confusing coordinates (particularly in 3D).
  • A line has infinitely many generating expressions :
    • Your solution and my model solution could look different although they both describe the same line.
    • Sometimes your computation will produce a direction vector with fractions as coordinates. You can avoid these by simply stretching the direction vector.
  • The same is true for planes in 3D, only more so!
  • When computing the intersection point between two lines (or a line and a plane, or two planes) with Gaussian elimination (as in Items 51 and 55) make sure you are not using the same parameter name twice! (Note how I used r and q in Item 55 for the second plane.)

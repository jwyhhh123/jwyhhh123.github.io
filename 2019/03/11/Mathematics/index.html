Gaussian Elimination
Base Case : ax = b. This can be solved directly: x = b/a.
General Case : There are n equations and each contains n unknowns.

  1. use the first equation to eliminate the first unknown in the remaing (n-1) equations.
  2. apply step 1 recursively.
  3. determine the last unknown and bring it back recursively.
  4. reduce each question to base case.

Special cases
Case 1 : contradictory equation
result : no solution

Case 2 : irrelevant equation
result : can be chosen freely

Case 3 : cannot use the first equation to eliminate.
solution : swap

Case 4 : cannot use any of the equations to solve the first unknown.
solution : General Gaussian Elimination

General Gaussian Elimination
Base Case 1 : Only one unknown is left over but there may be more than one equation. In this case we solve each equation independently and compare the answers; if they agree, then that is what we return. If they disagree, then the system has no solution (which we can communicate back by raising an exception).
Base Case 2 : Only one equation is left over but more than one unknown appears in it (let’s say m many). In this case m − 1 unknowns can be chosen freely and the other one is computed from the equation. (It does not matter which m − 1 unknowns are chosen and which is computed.)

Linear independence
Theorem 3 : A collection v1, …, vn of vectors is linear independent if ∑ a(i) * v(i) = 0 happens only in case a1 = a2 = a3 = … = an = 0.

  • the null vector 0 does not occur in the list v1, …, vn (because thi doesn’t generate anything)
  • none of the v(i) is a linear combination of the rest. (because this would be aredundancy)

proof:

if every element of an algebra V of vectors can be written as a linear combination of vector v1 … vn and if the v(i) of v1 … vn are linearly independent, then we call v1 … vn a basis of V.

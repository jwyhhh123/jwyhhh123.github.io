some numbers are not precise in a computer, for example pi, and the reason fot it is that we only have limit number of bits to represent a number. It is inevitable to lose some precision when we store floating point numbers.

IEEE754 Single-Precision Format(32 bits)
def : we introduce IEEE754 strategy to represent numbers in a computer.
 
sign bit: positive(0) or negative(1)
exponent: power
mantissa: fractional part of a number

Encoding strategy:

  1. record the sign bit
  2. convert the number part to binary
  3. convert the fractional part to binary
  4. concatenate 1. and 2.
  5. convert to scientific notation
  6. record the exponent
  7. record the mantissa

Density of floating point numbers

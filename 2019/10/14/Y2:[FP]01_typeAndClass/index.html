Types and Classes

WY J
2019-10-14 (Updated: 2019-12-25)
Year2/ Advanced functional programming (Haskell)

Types

def: a type is a collection of related values. For example Bool has types True and False.

In general, we introduce expression e :: T which means the expression e has type T.

Type safe: it is said to be safe as type errors, such as not 3, would never occur. Haskell programs are type safe as type inference precedes evaluation.

NOTE: in order to obtain the type of an expression we use command : type.

Classes

def: a class is a collection of types(methods)

NOTES: A class is like an interface in Java. ONLY newtype or data can implement a class using the keyword instance.

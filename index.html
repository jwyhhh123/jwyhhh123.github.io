Light Introduction to Requirements Engineering

post @ 2019-10-14

Objectives:

  • To introduce the concepts of user and system requirements
  • To describe functional and non-functional requirements
  • To explain how soft requirements may organised in a requirements ducument

Requirements analysis and its definition

def: the process of establishing what services are required and the constraints on the system’s operation and development. It states what the system is about.

Requirements engineering process:

  1. Feasibility study
  2. Requirements elicitation and analysis
  3. Requirements specification
  4. Requirements validation

NOTE:

  • Requirements(services and constraints) are generated during the process of engineering.
  • Requirements can be high-level abstract statement or can be a detailed functional specification.

General or detailed Requirements

“General” Statement:
this kind of statement shows how the system would achieve, interact with the user, other system, and environment…

examples:

  • The system contains all library items including books, newspaper, video… (service)
  • No item shall be removed from the library if it has not been recorded. (constraints)
  • All item shall have a bar code containing a uniqur reference number. (service)

“Detailed” Statement:
this kind of statement could be:

  1. Functional Requirements -> statements of the system’s functionality
  2. Non-Functional Requirements -> statements of the practical constraints or limitations within which the system must operate
  3. Implementation Requirements -> statements on how the system to be implemented

examples:

  • The system shall permit all users to search for an item by title, by author, by ISBN. (functional)
  • Borrowed items that are one day overdue shall cause a reminder e- mail to the borrower. (functional)
  • The system shall respond to a transaction requests from a user within 1.5 seconds. (non-functional)
  • When an item is borrowed or returned, it should be scanned through a card reader. (Implementation)

Computer Architecture

post @ 2019-10-06

Types of computer

Computers can be classified into:

  1. Application Specific Computer
  2. General Purpose Computer

The von Neumann Architecture

def: this is known as stored-programming computer in which no re-programming is required.

Von Neumann architecture consists of three main components:

  1. Central Processing Unit (CPU)
  2. Memory
  3. Input/Output interfaces

Process:

  1. read memory
  2. send to CPU and compute
  3. communicate with I/O interface
  4. (re)write data

NOTE: in this kind of architecture, CPU and Memory are seperated.

CPU

CPU is the head of a computing system. It includes:

  1. Control Unit (CU)
  2. Arithmetic and Logic Unit (ALU)

Arithmetic and Logic Unit (ALU)
def: ALU is the union of the circuits which perform mathematical or logical operations.

Control Unit (CU)
Control Unit performs the following tasks:

  1. Determines the order in which instructions should be executed
  2. Instructs ALU to perform the required operation
  3. Controls which Register to read and write
  4. Controls read and write of Memory
  5. Controls access of Input/ Output devices

NOTE_1: in order to know the type of an instruction(i.e. Read, Store, Operate…), Instruction decoder performs:

  • decodes instructions
  • understand the type of instruction
  • indicate the type to CU

NOTE_2: within the Control Unit, there is timing circuit. The timing of operations ensures step-by-step computation.

Registers
def: Registers are small storage which are located next to ALU. They are used to store temporary storage during computation. ALU can read and write to registers very fast.

Advantage: they improve the processing speed when compution needs to reuse some temporary data.
for example, to compute c = (a * b) + (a + b) + (b - a) we need to compute:

  1. c = a * b
  2. c = c + (a + b)
  3. c = c + (b - a)

In contrast, we save time from reading and writing c again and again.

def: The computer’s memory stores data and instructions. They are stored in binary.

Introduction to machine learning and k-NN

post @ 2019-06-02

Introduction to machine learning

def : machine learning focuses on studying and developing computational models capable of improving their performance with experience and acquiring knowledge on their own.
 
types of learning :

  1. supervised learning (监督学习)
  2. reinforcement learning (强化学习)
  3. unsupervised learning
  4. semi-supervised learning

Introduction to supervised learning

Predicative tasks : based on existing training data, we are aiming to learn models which are able to make predictions for new data.
 
Components :

  1. training data
  2. machine learning algorithm
  3. predictive model

Data format : (x, y)

  • x the input attributes : x = (x1, x2, x3 …, xn)
  • y the output attributes : we focus on ONE output.

Example :

Types of input attributes :

  • Numerical : 123
  • Ordinal : {low, medium, high}
  • Categorical : {UK, CHN, US}

Classification && Regression

Classification :

  • the output attributes are categories/ classes
  • the learning model is called : Classifier

Regression :

  • the output attributes are numerical values

NOTE : k-nn is dealing with classification and regression.

Example application

1) Estimation of the effort required to develop a software project.

  • input : programming language; team expertise, estimated size, development type…
  • output : person-hours / months
    2) Credit card approval
  • input : age; salary; number of years holding a bank account…
  • output : yes / no
    3) Breast cancer prediction
  • input : clump thickness; uniformity of cell size, cell thickness…
  • output : yes / no

NOTE : 监督学习实际上是给当前模型输入已知的数据然后来分析新的数据。

K-NN (K-Nearest Neighbours)

k : how many neighbors to choose for classification.
 
classification problems : the test data (x,?) is classified to the majority among the output attributes of neighbour training data.
 
Numerical problems : the test data is computed by the average among the output attributes of neighbour training data.
 
Algorithm :

  1. give the input attributes of a new data (x,?) that we want to predict
  2. give k, the neighbors we are looking for.
  3. For each training example (x’, y’) :
    • compute distance(x,x’) = root{ Σ(for all i) sum[(xi - x’(i))^2] }
    • sort all of the distances computed in ascending order by a data structure
    • return first k data as the nearest neighbors
  4. return either the majority vote or the average.

Normalisation of numeric input attributes

Problem : Different numeric input attributes may have different scales. It will affect the Euclidean distance. for example, if x1 is in [0, 10] and x2 is in [100, 10000], then x2 will influence the distance more.

  • if new training data are loaded, then we need to take them into consider as well. We then require to de-normalise and re-normalise the data (including test data).
  • this process is to minimize errors.

Advantages && Disadvantages

Adv :

  1. training is simple and fast, just normalise and store the training data

Disadv :

  1. require large memory space to store all data
  2. making prediction is slow, we have to sort the whole data set in order to find k neighbors. (the algorithm is slow)

Application

  • program effort estimation
  • movies classification

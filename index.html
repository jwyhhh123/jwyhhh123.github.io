Light Introduction to Requirements Engineering

post @ 2019-10-14

Objectives:

  • To introduce the concepts of user and system requirements
  • To describe functional and non-functional requirements
  • To explain how soft requirements may organised in a requirements ducument

Requirements analysis and its definition

def: the process of establishing what services are required and the constraints on the system’s operation and development. It states what the system is about.

Requirements engineering process:

  1. Feasibility study
  2. Requirements elicitation and analysis
  3. Requirements specification
  4. Requirements validation

NOTE:

  • Requirements(services and constraints) are generated during the process of engineering.
  • Requirements can be high-level abstract statement or can be a detailed functional specification.

General or detailed Requirements

“General” Statement:
this kind of statement shows how the system would achieve, interact with the user, other system, and environment…

examples:

  • The system contains all library items including books, newspaper, video… (service)
  • No item shall be removed from the library if it has not been recorded. (constraints)
  • All item shall have a bar code containing a uniqur reference number. (service)

“Detailed” Statement:
this kind of statement could be:

  1. Functional Requirements -> statements of the system’s functionality
  2. Non-Functional Requirements -> statements of the practical constraints or limitations within which the system must operate
  3. Implementation Requirements -> statements on how the system to be implemented

examples:

  • The system shall permit all users to search for an item by title, by author, by ISBN. (functional)
  • Borrowed items that are one day overdue shall cause a reminder e- mail to the borrower. (functional)
  • The system shall respond to a transaction requests from a user within 1.5 seconds. (non-functional)
  • When an item is borrowed or returned, it should be scanned through a card reader. (Implementation)

MacOS/ basic commands

post @ 2019-06-18

终端(Terminal)是可以进行人机交互的Input/Output程序。当下主流操作系统 i.e. MacOS, Linux使用虚拟终端(Virtual Terminal), 并通过指令(commands)来实现一些方便用户的操作。
 
这里介绍MacOS环境下的终端指令。需要注意的是MacOS和Linux由Unix操作系统派生，所以终端指令较为相似。

Terminology

Directories: 目录; 对于计算机，将目录的路径理解为指针(pointers)更为合适。

更直观地，我们可以使用树(Tree)表示:

指令格式: command [options…] [arguments…]
command: 指令名称
options: 修饰(modifiers); 通常为 “-“
arguments: 参数(parameters); 通常为文件名称

NOTE : 中括号可以省略。

man 使用手册

def: 对于任意一条命令，我们也许不知道它的具体功能和使用方法。man命令就像一本字典，我们可以查询一项命令的用户手册。
format: man [-k] name
 
example: man pwd

List of some commands

pwd 打印当前路径

cd 更改路径

ls 列出目录下文件

mkdir 新建文件夹

rmdir 删除文件夹

cp 复制文件

find 查找文件

cat 打印文件内容

more cat2.0

wc 字数统计

mail 邮件

talk 聊天

say 让电脑说话

cal 日历

date 日期

time 时间

Informed Search

post @ 2019-06-02

Informed Search

def : unlike uninformed searching algorithm, informed search carefully choose which node should be visited first in order to minimise the cost.
 
Heuristic functions(启发函数) : h(x) estimate the cost of the cheapest path from the node reached to a goal node.

A star Search

def : best-first informed search algorithm
 
Evaluation function : f(node) = g(node) + h(node)
g(node) : the cost from root node to current node.
 
Algorithm :

  1. visit the node with the smallest f(node) first
  2. placing its children in the frontier
  3. do not place children in the frontier if they are already visited or are in the frontier.
  4. when found the goal node, stop.

Consistency

Heuristic is said to be consistent if h(n) satisfies : h(n) ≤ cost(n, n’) + h(n’) for every edge (n, n’) of the graph. In such a case, A star is complete and admissible.
 
Time.compx : usually exponential
Space.compx : usually exponential

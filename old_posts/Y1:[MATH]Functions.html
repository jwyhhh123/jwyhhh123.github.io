Set-theoretic definition of functions

From a binary relation R ⊆ A × B, we define a function by satisfying two conditions:

  1. Definedness : ∀a ∈ A ∃b ∈ B. (a, b) ∈ R
  2. Single valueness : ∀a ∈ A ∀b,b’ ∈ B. (a,b) ∈ R ∧ (a,b’) ∈ R -> b = b’

Notations (given A x B) :

  • domain : A
  • co-domain : B
  • range : elements in B which are the results of F(A)

Representation : f : A -> B
 
Partial function : a relation that is single valued but not defined for all inputs.

Injectivity

def : a function that will give different output for different input is called injective (one-to-one).
 
formula : ∀a,a’ ∈ A. f(a) = f(a’) -> a = a’
 
Error correcting codes : coding theory where input “data vectors” are mapped to much longer “code words” (which are again tuples of bits). If this is done cleverly, then small errors that change the code word (for example, during transmission over long distances) can be discovered and repaired.
 
NOTES : non-injective examples are mps and jpeg.
 
Pigeon hole principle : if domain A has more elements than co-domain B, then no function from A to B can be injective.
 
Theorem 11 :

  1. if card(A) <= card(B) and card(B) <= card(A) then card(A) = card(B)
  2. for any two sets A and B, we have card(A) <= card(B) or card(B) <= card(A)

NOTE : see the picture above, it is injective.

Surjectivity

def : a function for which the range is all of the co-domain is called surjective (onto).
 
formula : ∀b ∈ B ∃a ∈ A. f(a) = b
 
Theorem 12 : assuming A != 0/, card(A) <= card(B) iff there is surjective function from B to A.
 
examples : project a 3-D object to 2-D picture.

NOTE : 两个不同的input可以映射到同一个output上。每一个input和每一个output都需要被覆盖。

Bijectivity

def : a combination of injectivity and surjectivity.

example : cryptographic encodings

Forward image and pre-image; inverse function

Given X is a subset of domain A, and Y a subset of co-domain B.

  • forward image(像) : f[X] = {b ∈ B | ∃a ∈ X. f(a) = b}
    -pre-image(原像) : f^-1[Y] = {a ∈ A | f(a) ∈ Y}

NOTE : y = x^2 是某种程度上的双射。f(-2)的像是4，然而f^-1[4]的原像是{2,-2}。

Counting functions

if A is a finite set with n members, B is a finite set with m members.
number of injective functions from A to B : m > n

NOTE : if n > m then result of this is 0 as there must have (m-k) = 0 (pigeon hole principle)
 
number of bijective functions from A to B : m = n

NOTE : the number of functions N -> N is uncountable.

Beyond Regular Languages

post @ 2019-03-11

Non-regular languages

def : we know there are some languages which are not regular because there are uncountably many languages and countably many regexps.
 
prove that a language is not regular: example of well-bracketed words

  1. if L is regular, then there is a DFA(X, p, δ, Acc) recognizes it.(K’s T)
  2. suppose a, b ∈ L and m < n.
  3. let Xm be the state reached when we start from p and read a^m.
  4. let Xn be the state reached when we start from p and read a^n.
  5. if starting from Xm and reading b^m then we reach a Acc.
  6. if starting from Xn and reading b^m then we reach a nonAcc.
  7. So Xn != Xm
  8. So there are infinitely many states.
  9. we raise a contradiction. Hence the language is not regular.

NOTE : 我们通过这种方式建立了一个DFA返回yes或no，然而论证中出现infinite，和我们的finite矛盾。

Functions

post @ 2019-03-11

Set-theoretic definition of functions

From a binary relation R ⊆ A × B, we define a function by satisfying two conditions:

  1. Definedness : ∀a ∈ A ∃b ∈ B. (a, b) ∈ R
  2. Single valueness : ∀a ∈ A ∀b,b’ ∈ B. (a,b) ∈ R ∧ (a,b’) ∈ R -> b = b’

Notations (given A x B) :

  • domain : A
  • co-domain : B
  • range : elements in B which are the results of F(A)

Representation : f : A -> B
 
Partial function : a relation that is single valued but not defined for all inputs.

The many faces of functions

in computer science, we would represent a function like :

Injectivity

def : a function that will give different output for different input is called injective (one-to-one).
 
formula : ∀a,a’ ∈ A. f(a) = f(a’) -> a = a’
 
Error correcting codes : coding theory where input “data vectors” are mapped to much longer “code words” (which are again tuples of bits). If this is done cleverly, then small errors that change the code word (for example, during transmission over long distances) can be discovered and repaired.
 
NOTES : non-injective examples are mps and jpeg.
 
Pigeon hole principle : if domain A has more elements than co-domain B, then no function from A to B can be injective.
 
Theorem 11 :

  1. if card(A) <= card(B) and card(B) <= card(A) then card(A) = card(B)
  2. for any two sets A and B, we have card(A) <= card(B) or card(B) <= card(A)

NOTE : see the picture above, it is injective.

Surjectivity

def : a function for which the range is all of the co-domain is called surjective (onto).
 
formula : ∀b ∈ B ∃a ∈ A. f(a) = b
 
Theorem 12 : assuming A != 0/, card(A) <= card(B) iff there is surjective function from B to A.
 
examples : project a 3-D object to 2-D picture.

NOTE : 两个不同的input可以映射到同一个output上。每一个input和每一个output都需要被覆盖。

Bijectivity

def : a combination of injectivity and surjectivity.

example : cryptographic encodings

Forward image and pre-image; inverse function

Given X is a subset of domain A, and Y a subset of co-domain B.

  • forward image(像) : f[X] = {b ∈ B | ∃a ∈ X. f(a) = b}
    -pre-image(原像) : f^-1[Y] = {a ∈ A | f(a) ∈ Y}

NOTE : y = x^2 是某种程度上的双射。f(-2)的像是4，然而f^-1[4]的原像是{2,-2}。

Composing functions

Representation of composing function f : A -> B with g : B -> C

in math and computer science we represent this from right to left : g◦f

Counting functions

if A is a finite set with n members, B is a finite set with m members.
number of injective functions from A to B : m > n

NOTE : if n > m then result of this is 0 as there must have (m-k) = 0 (pigeon hole principle)
 
number of bijective functions from A to B : m = n

NOTE : the number of functions N -> N is uncountable.

Gaussian Elimination

Base Case : ax = b. This can be solved directly: x = b/a.
General Case : There are n equations and each contains n unknowns.

  1. use the first equation to eliminate the first unknown in the remaing (n-1) equations.
  2. apply step 1 recursively.
  3. determine the last unknown and bring it back recursively.
  4. reduce each question to base case.

Special cases

Case 1 : contradictory equation
result : no solution

Case 2 : irrelevant equation
result : can be chosen freely

Case 3 : cannot use the first equation to eliminate.
solution : swap

Case 4 : cannot use any of the equations to solve the first unknown.
solution : General Gaussian Elimination
answer : the first unkown can be chosen freely

General Gaussian Elimination

Base Case 1 : Only one unknown is left over but there may be more than one equation. In this case we solve each equation independently and compare the answers; if they agree, then that is what we return. If they disagree, then the system has no solution (which we can communicate back by raising an exception).
Base Case 2 : Only one equation is left over but more than one unknown appears in it (let’s say m many). In this case m − 1 unknowns can be chosen freely and the other one is computed from the equation. (It does not matter which m − 1 unknowns are chosen and which is computed.)

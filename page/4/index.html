Regular expressions

We use regular expression to solve :

  1. Matching problem
  2. Searching problem

Σ : set of characters
Σ^* : set of all words
L : natural languages(passwords, email addresses)

NOTE : L ⊆ Σ^*

Notations :

  • The regexp a accepts ONLY the word a.
  • The regexp b accepts ONLY the word b.
  • ε accepts ONLY the empty word ε.
  • if E and F are regexps, then EF is a concatenation of words to be accepted.
  • if E and F are regexps, then E|F accepts either E or F.
  • E^* accepts words of E occuring zero or more times.
  • The regexp Nothing does not accept any word.

laws of precedence : E|F < EF < ()^*

Regular and Irregular language

Regular languages :
any language L ⊆ Σ^* that can be represented in this way is said to be regular.
 
Irregular languages :
we can ONLY show the existence as there are uncountablly many languages and countablly many regexps.

NOTES :

  • the complement of a regular language is still regular : a regexp that NOT accepts c(bb|ca)^*
  • the intersection of two regular languages is still regular : a regexp accepts both cc(bb|ca)^* and c(bbbb|cca)^*

Decidability

def : problem is decidable when there is some program that can determine the answer of yes and no.
 
to show a problem is decidable, introduce a program to do so.
to show a problem is undecidable : reduce to the halting problem or, use rice’s theorem.

examples :

  1. there is a program boolean equality(String E, String F){} that can determine if two regexps are equal, so this problem is decidable.
  2. for every regexp E, matching problem for E is decidable.

Introducing Automata

DFA(Deterministic Finite Automaton) :

  • a finite set X of states
  • an initial state p ∈ X
  • a transition function δ: X × Σ → X
  • a set of accepting states Acc ∈ X; Acc is indicated by a double ring.

“Deterministic” because the initial state and the result of each transition are specified. “Finite” because the set of states is finite.

partial DFA : δ is a partial function, meaning there are undefined transitions

NOTES :

  • DFA includes all possible transitions
  • partial DFA includes NOT all transitions.
  • turn a partial DFA into DFA: add an error state which takes all undefined transitions and transitions to itself.

Non-deterministic Automata

difference:

  1. NDFA can have 1 and more initial states
  2. from a given state, when a (or any other character) is the input, there can be several possible next states. Example : 1 -> a -> 2 and 1 -> a -> 3

Determinization :

  1. The initial “state” is the set of all the initial states of the NDFA.
  2. A “state” is accepting when it contains an accepting state of the NDFA.
  3. From a given “state”, when we input a character, we collect all the possible next states.

ε-transitions

def : transition taking empty word ε
εNDFA : nondeterministic automaton with ε-transitions
 
Convert from εNDFA to NDFA :

  1. remove ε-transitions
  2. add new transitions
  3. determine new Acc
  4. remove non reachable states

NOTE : during converting, we do not remove any reachable state.

How to add sets

sum A + B :

  • inl a = (0, a)
  • inr b = (1, b)

example : {3,5,6} + {3,6,9,28}

NOTE : A + 1 where 1 represents a singleton set. We use this to convert a partial DFA into DFA by adding and error state.

Kleene’s Theorem

theorem 1 : for a language L ⊆ Σ^* , the following are equivalent:

  1. L is regular
  2. The matching problem for L can be solved by a DFA.

We ONLY study from 1. to 2. : if L is regular, then there is a DFA that can solve the matching problem for L.
 
We are now, based on Kleene’s theorem, able to convert a regular language L into DFA by:

  1. transfer L into εNDFA
  2. transfer εNDFA into NDFA (remove ε-transition)
  3. transfer NDFA to DFA (determinize)

Using Kleene’s Theorem

Although regexps and finite automata are equivalent, it’s often easier to operate on automata. For example, we asked: is the complement of a regular language regular? That’s not obvious for regexps, but using (total) DFAs, it is clear: just replace ac- cepting states by non-accepting ones and vice versa.
 
This implies that the intersection of regular languages L and M is regular, because M ∩ L = L ∪ M . But we can also see this directly. Say L is recognized by automaton A, and M by B. If you’re given a word w and you want to know whether it’s in L ∩ M, passing through w just once, you can simultaneously keep track of where you are in A and where you are in B, as you read through w. So your state at any time is a pair (x, y), where x is a state in A, and y is a state in B. This immediately gives you a DFA.
 
For example, say you want a DFA for the password question in the last handout. You can make a DFA that determines whether a word has at least 3 characters, and another that determines whether it has a letter, and another that determines whether it has a digit. Then we want a DFA for the intersection of these languages. Kleene’s theorem tells us that there must be some corresponding regexp.

Language equivalence

def : According to Kleene’s Theorem, we can check if two regexps are language equivalent by checking their DFAs.
 
Procedure :

  1. starting from the initial states of each automaton
  2. traverse from each word from Σ. This will give us a tree structure
  3. stop when given state is visited.
  4. if language inequivalent, then there is the case when one reaches an accepting state and the other reaches a rejecting state.
  5. if language equivalent, then all frontiers are stopped.

Minimizing a DFA

def : make a DFA as small as possible
 
Procedure :

  1. remove unreachable states.
  2. identify states that are language equivalent.

To prove a DFA is minimal :

  1. show each state is reachable, by giving a word that covers all states. (1. is satisfied)
  2. show that any two distinct states are not language equivalent, by giving a word that one accepts and the other rejects. (2. is satisfied)

Church’s Thesis

def : any decision problem on words that can be solved by an algorithm can be solved by a turing machine.
 
other version for functions f : Σ^ → Σ^ : any functions on words that can be computed by an algorithm can be solved by a turing machine..
 
computable : if there is a turing machine can return f(word) to a given word
decidable : if there is a turing machine can return yes or no to a given argument
 
examples :

  • A function f : N -> N is computable when there is a turing mahchine that starting from a natural number, written in binary, halts with the number f(n) in binary. By Church’s thesis, it is computable when there is some program that when given a number n, it returns f(n).
  • Ambiguity of a context free grammar is undecidable. This means there is no turing machine can return yes if the grammar over {a, b, |, ::=, nowhere, _} is ambiguous an no if is unambiguous. By Church’s thesis, there is no program that when given a grammar over the set of {a, b, |, ::=, nowhere, _} returns true if the grammar is ambiguous and false if not.

  • natural numbers, Turing machines, Java programs and context free grammars can be encoded as words
  • words, Turing machines, Java prorgrams and context free grammars can be encoded as natural numbers.

Reduction of a problem to another problem

def : when given problem P and Q, suppose that we show how to solve Q using a black box that solves P. Then is called reducing problem Q to problem P .

  • if Q is undecidable, P is then undecidable

Preliminary: important summation

Runing time of a program

Complexity

We don’t usually work out the precise running time of a program; indeed we don’t usually have the information needed to do so. Instead we work out the complexity, which is a kind of rough estimate of the running time that ignores two things:

  1. small arguments
  2. constant factors

Big-Oh

def : let f(n) be the running time for an argument of size n. let g(n) be another such function. there is the case when f ∈ O(g) given natural number M and constant factor C, such that for all n >= M we have f(n) <= C * g(n)
 
logical symbol : ∃M. ∃C. ∀n >= M. f(n) <= C × g(n)
rearrange the inequality : f(n)/g(n) <= C
 
NOTE : C is the upper bound when it is the maximum of f(n)/g(n).
 
we use big-Oh to solve complexity problems :

  • The running time of my program is 3n2 + 9n + 8. Is this in O(n2)? In O(n)? In O(n3)?
  • The running time of my program is 5n for n < 1000, and 3n2 +9n+8 for n 􏰂< 1000. Is this in O(n2)? In O(n)? In O(n3)?
  • The running time of my programn is 5n. Is this in O(n2)? In O(n)? In O(n3)?
  • On an argument of size n, I first run a program whose running time is in O(n2), and then run a program whose running time is in O(n3). Show that the total running time is in O(n3).

NOTE : exponential growth is faster than polunomial growth.

Polynomial Time

def : The program is polynomial time when there is k such that f ∈ O(nk). In detail, it is polynomial time when there is k and M and C such that if n 􏰂 M then f(n) 􏰁 C × nk.

Space Complexity

We can study the space usage of a program in a similar way. For example, suppose my program, on an argument of size n, uses 8 * n^2 + 5 bytes of memory. We then say that the space usage is in O(n^2).

Polynomial time algorithms

P : a polynomial time algorithm has running time in O(n^k). A polynomial time algorithm is said to be feasible; an algorithm that’s not polynomial time is said to be infeasible.
 
some example algorithms are :

  • A constant time algorithm has running time in O(1).
  • A linear time algorithm has running time in O(n).
  • A quadratic time algorithm has running time in O(n2).
  • A cubic time algorithm has running time in O(n3).

NP Search problems

A search problem is a relation from the set of words(problem instances) to the set of words(solutions).
 
NP : an NP search problem is a search problem wehere solutions have polynomial size in n, and can be checked in a time polynomial in n. Here n is the size of the problem instance.

  • solutions have size polynomial in n : for a problem instance of size n, all words that are solutions have size <= p(n). (where p(n) is some O())
  • solutions can be checked in a time polynomial in n : given a problem instance A and a solution B, there is an alogorithm that checks whether it is <= q(n). (where q(n) is some O())

from these two steps, we can find a solution in exponential time. As the number of words(for example 0-9; a-z) is r, there are at most (r + 1)^p(n). And, each word can be checked in q(n) so overall we have (r + 1)^p(n) * q(n) steps to find a solution, which is exponential time.
 
“NP = P” question : can every NP seach problem be solved in polynomial time?

example : sudoku (NP)

  1. solutions have size polynomial in n…
  2. solutions can be checked in a time polynomial in n…
  3. we can find a solution in exponential time…

example : factorization (NP)

  1. solutions have size polynomial in n…
  2. solutions can be checked in a time polynomial in n…
  3. we can find a solution in exponential time…

Properties of Code

post @ 2019-03-11

Undecidability by reduction

def: to prove other undecidable problem we can reduce the halting problem to it.
Because if reducing Q to P we have

  • if P is decidable then Q is decidable
  • if Q is undecidable then P is undecidable

example: X is a undecidable property when X halts and contains more occurances of a than b
proof:

  1. reduce the halting problem to X.
  2. for any program C of type void. let F(C) be the same code as C with an extra comment line which gives more occurances of a than b.
  3. if C halts, then F(C) is X
  4. if C does not halts, then F(C) is not X
  5. we cannot test whether C halts, so we cannot decide whether it is X.
  6. as C is undecidable, F(C) is undecidable
  7. so X is undecidable

Semantic and non-semantic properties

semantic properties:

  • Does this software print only polite words?
  • Does this software provide good advice to all users, and excellent advice to premium users?
  • Does this software, when supplied with two positive integers, always return the highest common factor?

to test whether a given property is a semantic property or not:

  • to show not semantic, give two programs one has the property and one has not.
  • to show semantic, there is no program has different properties.

Rice’s Theorem

def: any semantic property of code that holds in some case and does not hold in some case is undecidable.

  3. ¬ not <--> -
  4. Falsity <--> 0/

Boolean algebra

Relations

General relations

def : if A1, A2, …, An are sets and R is a subset of A1 x A2 x …x An, then R is called a (n-ary) relation.
 
NOTE : the elements of A1 ×A2 ×…×An are n-tuples (x1,x2,…,xn). Also, note that any subset is allowed; there are no conditions for a subset to be called a relation.
 
Example : 3-ary relation given A1 : the set Σ∗ of strings, and A2, A3: sets of N

NOTE : we are looking ar the special case of 2-ary(binary) relations on a single set. To explain, we look at subsets of A x A
 
Directed graph representation :

  • vertices : elements in A
  • edges : relations/ subsets of A

Reflexivity

def : We call a binary relation R ⊆ A × A reflexive if it contains all pairs (x, x) for x an element of A.
 
R ⊆ A x A is reflexive = ∀x ∈ A. (x, x) ∈ R
 
if given a relation R that is not reflexive, then we can make it reflexive by adding the missing edges :
refl-closure(R) = R ∪ {(x, y) ∈ A x A | x = y}
 
Irreflexive relation : relation that contains no edge pointing to itself

Symmetry

def : A binary relation R ⊆ A × A is called symmetric if with every pair (x, y) ∈ R we also have (y, x) ∈ R.
 
R ⊆ A × A is symmetric = ∀(x,y) ∈ A×A. (x,y) ∈ R -> (y,x) ∈ R
 
if given a relation R that is not symmetric, then we can make it symmetric by giving:

  1. inverse relation : R^(-1) = {(x,y) ∈ A x A | (y,x) ∈ R}
  2. symm-closure(R) = R ∪ R^(-1)

Anti-symmetric : there is no a and b connecting each other. While reflexivity is allowed.

Transitivity

def : A binary relation R ⊆ A × A is called transitive if whenever (x, y) ∈ R and (y, z) ∈ R then also (x, z) ∈ R.
 
R ⊆ A × A is transitive = ∀x,y,z ∈ A.(x,y) ∈ R and (y,z) ∈ R -> (x,z) ∈ R
 
if given a relation R that is not transitive, then we can make it transitive by given :

  1. composition of binary relations : R;S = {(x,z) ∈ A x A | ∃y ∈ A. (x, y) ∈ R and (y, z) ∈ S}
  2. trans-closure(R) = R ∪ (R;R) ∪ (R;R;R) ∪ …

NOTE : this idea is like finding a path from liverpool to birmingham, there are lot of paths we can choose.

Two packages of properties

Order relation :
A relation R ⊆ A × A is called an order relation if it is reflexive, anti-symmetric, and transitive.

Equivalence relation :
A relation R ⊆ A × A is called an equivalence relation if it is reflexive, symmetric, and transitive.

Equivalence relations

symbol : ≈ (congruence modulo m is commonly written as ≡ or ≡m).
equiv-closure(R) = trans-closure(symm-closure(refl-closure(R)))

Equivalence relations lead to classifications

Equivalence classification : [a]≈ = {x ∈ A| x ≈ a} where a is an element of A.
 
Theorem : if ≈ is an equivalence relation on a set A, and a, b are elements of A, then always one of the following two situations is true :

  1. a ≈ b and [a]≈ = [b]≈
  2. a !≈ b and [a]≈ ∩ [b]≈ = 0/

NOTE : this means class [a]≈ and class [b]≈ are disjointed.
